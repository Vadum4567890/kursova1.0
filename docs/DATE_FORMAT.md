# Формат дат в API

## Підтримувані формати дат

API приймає дати в різних форматах, які розпізнає JavaScript `new Date()`:

### Рекомендований формат (ISO 8601):
```
YYYY-MM-DD
```
**Приклади:**
- `2024-01-01`
- `2024-12-31`
- `2020-08-30`

### Альтернативні формати (також працюють):
```
DD.MM.YYYY    → 30.08.2020
DD/MM/YYYY    → 30/08/2020
MM-DD-YYYY    → 08-30-2020
YYYY/MM/DD    → 2020/08/30
```

### Повний формат з часом (ISO 8601):
```
YYYY-MM-DDTHH:mm:ss.sssZ
```
**Приклади:**
- `2024-01-01T00:00:00.000Z`
- `2024-12-31T23:59:59.999Z`

## Використання в Swagger UI

У Swagger UI використовуйте формат **YYYY-MM-DD**:

✅ **Правильно:**
- `2024-01-01`
- `2024-12-31`

❌ **Неправильно (може не спрацювати):**
- `30.08.2020` (крапка як роздільник)
- `30/08/2020` (слеш як роздільник)

## Приклади запитів

### cURL:
```bash
curl -X 'GET' \
  'http://localhost:3000/api/reports/financial?startDate=2024-01-01&endDate=2024-12-31' \
  -H 'accept: */*'
```

### JavaScript:
```javascript
const startDate = '2024-01-01';
const endDate = '2024-12-31';
const url = `http://localhost:3000/api/reports/financial?startDate=${startDate}&endDate=${endDate}`;
```

## Важливо

- Дати без часу інтерпретуються як **00:00:00 UTC**
- Для локального часу використовуйте повний формат з часом
- Якщо формат не розпізнається, API поверне помилку 500
